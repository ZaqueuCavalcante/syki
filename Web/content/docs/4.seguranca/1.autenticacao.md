## Login padrão

O acesso é feito com **e-mail e senha**. As senhas são armazenadas com hash bcrypt e nunca trafegam em texto puro.

## Autenticação em dois fatores (2FA)

O 2FA adiciona uma camada extra de segurança usando um aplicativo autenticador (Google Authenticator, Authy, etc.).

Para ativar:

1. Acesse **Configurações > Segurança**
2. Clique em **Configurar 2FA**
3. Escaneie o QR code com seu aplicativo
4. Insira o código de 6 dígitos para confirmar

::callout{type="info"}
Recomendamos que todos os usuários com acesso administrativo ativem o 2FA.
::

## Magic Link

O magic link permite login sem senha via e-mail. Um link de acesso único é enviado ao endereço cadastrado e expira em 15 minutos.

## SSO (Single Sign-On)

Para instituições com identidade corporativa, o Estud suporta login único (SSO) via OpenID Connect, com integração pronta para Azure AD, Google Workspace, Okta, Auth0 e provedores OIDC customizados. Configure em **Segurança > SSO**.

## Perfis e permissões

O controle de acesso é baseado em perfis. Cada perfil tem um conjunto de permissões granulares que determinam o que o usuário pode ver e fazer na plataforma.
